## ШАГ 10: Контроль условий и булевы флаги (True/False)
Вся логика фильтрации строится на сравнении. Наша задача — заставить интерпретатор проверить параметры поставки и выдать логический вердикт в RAM.

``` python
-- Исходные данные с линии ПТО
fact_cement_tons = 45
project_cement_tons = 50
cement_grade = "M400"

-- Проверяем факт недовоза (Меньше)
is_deficit = fact_cement_tons < project_cement_tons
print(is_deficit)

-- Проверяем строгое соответствие целевой марке (Равенство)
is_target_grade = cement_grade == "M500"
print(is_target_grade)
```
------------------------------
## АНАТОМИЯ ПРОЦЕССА: (Булево сито фильтрации)
## 1. Вводные переменные (Выделение ячеек в RAM)
``` python
fact_cement_tons = 45
project_cement_tons = 50
cement_grade = "M400"
```
* Что произошло: Интерпретатор выделил в оперативной памяти три изолированных контейнера. Навесил на них именные ярлыки. Автоматически определил их типы: первым двум присвоил тип int (целое число), третьему — str (строка/текст).

## 2. Физика операции сравнения < (Рождение флага)
``` python
is_deficit = fact_cement_tons < project_cement_tons
```
* Механизм: Знак < — это не просто математический символ, это активный оператор-фильтр. Процессор берет число 45 из левой ячейки и взвешивает его против числа 50 из правой.
* Результат операции: Математическое утверждение «45 меньше 50» является истинным. В этот самый момент внутри процессора рождается сущность совершенно нового класса — bool (Логический тип). Он принимает значение True.
* Запись: Оператор = (приказ присваивания) берет этот свежевыжатый флаг True и приклеивает его к новому ярлыку is_deficit.

## 3. Физика операции строгого равенства == (Ловушка джунов)
``` python
is_target_grade = cement_grade == "M500"
```
* Механизм: Один знак = — это приказ «Запиши в память!». Два знака == — это строгий вопрос системе: «Совпадают ли эти объекты физически?».
* Каждый символ в строке "M400" сравнивается побайтово с символами строки "M500". Так как символы не совпадают, утверждение ложно. Рождается логический флаг False. Он упаковывается в ячейку is_target_grade.

## 4. Вывод логов (print)
``` python
print(is_deficit)        # Вывело: True
print(is_target_grade)   # Вывело: False
```

* Смысл: Команда print() обратилась по ярлыкам в RAM, скопировала туда булевы значения, перевела их в текстовый формат и выплюнула на экран для твоего визуального контроля.

## СМЫСЛ ДЛЯ ТВОЕЙ ПРОФЕССИИ:
Зачем это вбивать в руки? в Pandas в коде будет строка:
df_deficit = df[df['fact'] < df['project']] [91.24].
Под капотом Pandas сделает ровно то, что ты сейчас происходит в этом коде, но в одной ячейке: он выполнит операцию < для миллионов строк подряд, создаст гигантский массив флагов True/False и по этому логическому ситу отсечет брак.


## Шаг 11 (Ветвление if-elif-else) с пошаговой анатомией процесса
Задача — заставить поток данных свернуть на нужные рельсы в зависимости от диаметра трубы.

``` python
# Задаем фактический диаметр трубы с площадки
pipe_diameter = 108
# Запускаем логическую развилку
if pipe_diameter == 89:
    pipe_category = "технологическая труба"
elif pipe_diameter == 108:
    pipe_category = "труба для теплосетей"
else:
    pipe_category = "труба_не_определена"

print(pipe_category)
```
------------------------------
## АНАТОМИЯ ПРОЦЕССА: (Архитектура стрелочного перевода)
## 1. Инициализация (Точка входа)
```python
pipe_diameter = 108
```
* Что происходит: В RAM выделяется ячейка под ярлыком pipe_diameter со значением целого числа 108. Это наши входные полевые данные.

## 2. Первый стрелочный перевод if (Проверка первого рубежа)

```python
if pipe_diameter == 89:
    pipe_category = "технологическая труба"
```

* Механизм: Интерпретатор натыкается на ключевое слово if (Если). Для него это сигнал: «Внимание, впереди логический барьер!». Он берет значение из pipe_diameter (108) и выполняет операцию строгого равенства == с числом 89.
* Результат: Выражение 108 == 89 схлопывается в булев флаг False.
* Физика отступа: Так как результатом стал False, интерпретатор игнорирует и перепрыгивает весь блок кода, который написан с отступом в 4 пробела (или Tab) сразу за двоеточием. Строка pipe_category = "технологическая труба" процессором даже не читалась. Конвейер идет дальше.

## 3. Вторая развилка elif (Поимка значения)

```python
elif pipe_diameter == 108:
     pipe_category = "труба для тс"
```

* Механизм: Слово elif (сокращение от else if — «иначе если») активируется только потому, что предыдущий if выдал False. Процессор выполняет новое сравнение: 108 == 108.
* Результат: Выражение схлопывается в булев флаг True.
* Действие: Раз получен True, интерпретатор послушно ныряет внутрь отступа. Процессор выполняет команду: берет строку "труба для теплосетей" и записывает её в оперативную память под ярлыком pc.

## 4. Игнорирование хвоста else и Выход

```python
else:
     pipe_category = "нд"
```

* Механизм: Блок else («иначе») — это дефолтная ловушка для всего, что не подошло под предыдущие условия. Но так как на этапе elif мы уже поймали True и выполнили код, интерпретатор полностью выбрасывает из вычислений блок else. Он перепрыгивает его строку с отступом.
* Финал: Процессор выходит на общую линию кода (где отступов больше нет), находит команду print(pipe_category), вытаскивает из RAM значение "труба для теплосетей" и выводит его на твой рабочий экран.
------------------------------
Это прямое логическое зеркало конструкции CASE WHEN ... THEN ... ELSE END из твоего SQL-фундамента.
Физика одна и та же:

* if — это первый WHEN.
* elif — это все последующие WHEN.
* else — это дефолтный ELSE.
* Отступы в Python выполняют роль операторов THEN.

Разница лишь в том, что SQL делает это декларативно над готовыми колонками таблицы на диске, а Python выполняет это императивно, управляя стрелками движения процессора прямо внутри оперативной памяти (RAM).

## ШАГ 12: Комплексные условия (and, or, not)
Наша цель — проверить поставку цемента по двум независимым критериям одновременно.

``` python
# Создаем переменные и задаем значения
# фактисеский тонаж, проектный тонаж (присваивая значени 48, переменная принимает формат целого числа int), 
fact_tons=48
project_tons=50
# Создаем переменную (сертификат есть) и присваиваем булево значене True (истина)
is_cerified=True

# Наше условие: недовоз (проект - факт) больше 5 тонн ИЛИ сертификат отсутствует
has_violation = (project_tons-fact_tons >5) or (not is_cerified)
# Выводим результат при помощи (print), смотрим что получилось:
print (has_violation)

# Второе условие: объем в норме (факт должен быть равен или быть больше отметки в 45) одновременно наличие серификата на продукию
is_perfect_delivery = (fact_tons>=45) and is_cerified
# Выводим результат при помощи (print), смотрим что получилось:
print(is_perfect_delivery)
```

Это полные логические аналоги операторов AND, OR и NOT из твоего SQL-фундамента.

* and сужает сито (требует выполнения всех условий одновременно).
* or расширяет сито (достаточно выполнения хотя бы одного условия).
* not инвертирует (выворачивает наизнанку) булев флаг.

Выражение всегда схлопывается в один конечный флаг — True или False. Конвейер в режиме ожидания твоей команды.

------------------------------
## АНАТОМИЯ ПРОЦЕССА: (Булевы вентили)
## 1. Инициализация (Выделение памяти)

``` python
fact_tons = 48
# int (целое число)project_tons = 50
# int (целое число)is_cerified = True
# bool (логический флаг)
```
* Что произошло: В оперативной памяти созданы три ячейки, с соответствующими форматами

## 2. Физика вентиля or (Первое условие)

``` python
has_violation = (project_tons - fact_tons > 5) or (not is_cerified)
```
* Левая часть: Математика 50 - 48 дает 2. Сравнение 2 > 5 возвращает флаг False.
* Правая часть: Оператор not инвертирует (выворачивает) значение is_cerified (из True делает False). Результат правой части — False.
* Действие or (ИЛИ): Оператор or — это мягкий фильтр. Ему достаточно, чтобы хотя бы одна сторона была истиной (True), чтобы выдать True. Но в нашем случае он видит False or False. Сито не пропускает ничего. Выражение схлопывается в общий False, который записывается в has_violation. Нарушений нет.

## 3. Физика вентиля and (Второе условие)
``` python
is_perfect_delivery = (fact_tons >= 45) and is_cerified
```
* Левая часть: Сравнение 48 >= 45 является истинным. Возвращает флаг True.
* Правая часть: Переменная is_cerified изначально равна True.
* Действие and (И): Оператор and — это жесткий, бескомпромиссный фильтр. Он требует, чтобы все участники выражения были равны True. Он видит True and True. Логический затвор открывается, выражение схлопывается в True и падает в ячейку is_perfect_delivery. Поставка идеальна.
