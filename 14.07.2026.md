## АНАТОМИЯ ПРОЦЕССА: ШАГ 2.1 (Булево сито фильтрации)
## 1. Вводные переменные (Выделение ячеек в RAM)
``` python
fact_cement_tons = 45project_cement_tons = 50cement_grade = "M400"
```

* Что произошло: Интерпретатор выделил в оперативной памяти три изолированных контейнера. Навесил на них именные ярлыки. Автоматически определил их типы: первым двум присвоил тип int (целое число), третьему — str (строка/текст) [91.24].

## 2. Физика операции сравнения < (Рождение флага)
``` python
is_deficit = fact_cement_tons < project_cement_tons
```

* Механизм: Знак < — это не просто математический символ, это активный оператор-фильтр. Процессор берет число 45 из левой ячейки и взвешивает его против числа 50 из правой.
* Результат операции: Математическое утверждение «45 меньше 50» является истинным. В этот самый момент внутри процессора рождается сущность совершенно нового класса — bool (Логический тип) [91.24]. Он принимает значение True [91.24].
* Запись: Оператор = (приказ присваивания) берет этот свежевыжатый флаг True и намертво приклеивает его к новому ярлыку is_deficit [91.24].

## 3. Физика операции строгого равенства == (Ловушка джунов)
``` python
is_target_grade = cement_grade == "M500"
```

* Механизм: Один знак = — это приказ «Запиши в память!» [91.24]. Два знака == — это жесткий вопрос системе: «Совпадают ли эти объекты физически?» [91.24].
* Каждый символ в строке "M400" сравнивается побайтово с символами строки "M500". Так как символы не совпадают, утверждение ложно. Рождается логический флаг False [91.24]. Он упаковывается в ячейку is_target_grade [91.24].

## 4. Вывод логов (print)
``` python
print(is_deficit)        # Вывело: True
print(is_target_grade)   # Вывело: False
```

* Смысл: Команда print() обратилась по ярлыкам в RAM, скопировала туда булевы значения, перевела их в текстовый формат и выплюнула на экран для твоего визуального контроля [91.24].

## АНАТОМИЯ ПРОЦЕССА: ШАГ 2.2 (Архитектура стрелочного перевода)## 1. Инициализация (Точка входа)
```python
pd = 108
```

* Что происходит: В RAM выделяется ячейка под ярлыком pd со значением целого числа 108. Это наши входные полевые данные [91.24].

## 2. Первый стрелочный перевод if (Проверка первого рубежа)
```python
if pd == 89:
    pc = "технологическая труба"
```

* Механизм: Интерпретатор натыкается на ключевое слово if (Если). Для него это сигнал: «Внимание, впереди логический барьер!» [0.1.2, 91.24]. Он берет значение из pd (108) и выполняет операцию строгого равенства == с числом 89 [0.1.2, 91.24].
* Результат: Выражение 108 == 89 схлопывается в булев флаг False [91.24].
* Физика отступа: Так как результатом стал False, интерпретатор игнорирует и перепрыгивает весь блок кода, который написан с отступом в 4 пробела (или Tab) сразу за двоеточием [0.1.2, 91.24]. Строка pc = "технологическая труба" процессором даже не читалась. Конвейер идет дальше.

## 3. Вторая развилка elif (Поимка значения)
```python
elif pd == 108:
    pc = "труба для тс"
```

* Механизм: Слово elif (сокращение от else if — «иначе если») активируется только потому, что предыдущий if выдал False. Процессор выполняет новое сравнение: 108 == 108 [0.1.2, 91.24].
* Результат: Выражение схлопывается в булев флаг True [91.24].
* Действие: Раз получен True, интерпретатор послушно ныряет внутрь отступа [0.1.2, 91.24]. Процессор выполняет команду: берет строку "труба для тс" и записывает её в оперативную память под ярлыком pc [0.1.2, 91.24].

## 4. Игнорирование хвоста else и Выход
```python
else:
    pc = "нд"
```

* Механизм: Блок else («иначе») — это дефолтная ловушка для всего, что не подошло под предыдущие условия [0.1.2, 91.24]. Но так как на этапе elif мы уже поймали True и выполнили код, интерпретатор полностью выбрасывает из вычислений блок else. Он перепрыгивает его строку с отступом.
* Финал: Процессор выходит на общую линию кода (где отступов больше нет), находит команду print(pc) [0.1.2, 91.24], вытаскивает из RAM значение "труба для тс" и выводит его на твой рабочий экран.
------------------------------
Это прямое логическое зеркало конструкции CASE WHEN ... THEN ... ELSE END из твоего SQL-фундамента.
Физика одна и та же:

* if — это первый WHEN.
* elif — это все последующие WHEN.
* else — это дефолтный ELSE.
* Отступы в Python выполняют роль операторов THEN.

Разница лишь в том, что SQL делает это декларативно над готовыми колонками таблицы на диске, а Python выполняет это императивно, управляя стрелками движения процессора прямо внутри оперативной памяти (RAM) [0.1.2, 91.24].

* and сужает сито (требует выполнения всех условий одновременно).
* or расширяет сито (достаточно выполнения хотя бы одного условия).
* not инвертирует (выворачивает наизнанку) булев флаг.
