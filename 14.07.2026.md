## АНАТОМИЯ ПРОЦЕССА: ШАГ 2.1 (Булево сито фильтрации)## 1. Вводные переменные (Выделение ячеек в RAM)

fact_cement_tons = 45project_cement_tons = 50cement_grade = "M400"


* Что произошло: Интерпретатор выделил в оперативной памяти три изолированных контейнера. Навесил на них именные ярлыки. Автоматически определил их типы: первым двум присвоил тип int (целое число), третьему — str (строка/текст) [91.24].

## 2. Физика операции сравнения < (Рождение флага)

is_deficit = fact_cement_tons < project_cement_tons


* Механизм: Знак < — это не просто математический символ, это активный оператор-фильтр. Процессор берет число 45 из левой ячейки и взвешивает его против числа 50 из правой.
* Результат операции: Математическое утверждение «45 меньше 50» является истинным. В этот самый момент внутри процессора рождается сущность совершенно нового класса — bool (Логический тип) [91.24]. Он принимает значение True [91.24].
* Запись: Оператор = (приказ присваивания) берет этот свежевыжатый флаг True и намертво приклеивает его к новому ярлыку is_deficit [91.24].

## 3. Физика операции строгого равенства == (Ловушка джунов)

is_target_grade = cement_grade == "M500"


* Механизм: Один знак = — это приказ «Запиши в память!» [91.24]. Два знака == — это жесткий вопрос системе: «Совпадают ли эти объекты физически?» [91.24].
* Каждый символ в строке "M400" сравнивается побайтово с символами строки "M500". Так как символы не совпадают, утверждение ложно. Рождается логический флаг False [91.24]. Он упаковывается в ячейку is_target_grade [91.24].

## 4. Вывод логов (print)

print(is_deficit)        # Вывело: True
print(is_target_grade)   # Вывело: False


* Смысл: Команда print() обратилась по ярлыкам в RAM, скопировала туда булевы значения, перевела их в текстовый формат и выплюнула на экран для твоего визуального контроля [91.24].
