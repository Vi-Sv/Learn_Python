## ЭПИЗОД I: АРХИТЕКТУРА И ИМПОРТ СЫРЬЯ

### Шаг 1. Проектирование изолированного цеха данных
Для соблюдения промышленной чистоты проекта в корне папки `D:\Portable_Data_Suite\` создана отдельная директория `data\`. Грязный файл `vagon_data.xlsx` заперт внутри неё. Это исключает засорение корня скриптов.

*   **Физический путь к сырью:** `D:\Portable_Data_Suite\data\vagon_data.xlsx`
### Шаг 2. Сканирование диска и заглатывание файла в RAM

```python
import pandas as pd

df_excel = pd.read_excel("D:/Portable_Data_Suite/data/vagon_data.xlsx")
```

#### Анатомический разбор роли каждого символа:
*   **`df_excel`** — Новое имя-ярлык для таблицы, которую мы сейчас создадим в RAM.
*   **`=`** — Оператор присваивания. Приказ: «Запиши результат ксерокопии файла в ярлык слева».
*   **`pd.`** — Обращение к шлюзу библиотеки Pandas.
*   **`read_excel(...)`** — Встроенный метод-сканер. Он не открывает файл для редактирования, а считывает его структуру, делает цифровую копию и разворачивает её в RAM, сразу закрывая связь с диском.
*   **`"D:/..."`** — Строковый аргумент внутри круглых скобок. Жесткий адрес-указатель, где именно на жестком диске робот должен искать физический файл.

### Шаг 3. Первичная разведка универсальным инструментом

```python
df_excel.head()
```
*   **`.head()`** — Универсальный метод отображения «Головы» таблицы. Он независим от среды (будет работать и в Jupyter, и на сервере Linux). Его обязанность — вывести на экран фиксированное количество первых строк (по умолчанию 5), сохранив строгую структуру.

### Шаг 4. Векторная зачистка текстового шума и регистров

```python
df_excel["cargo"] = df_excel["cargo"].str.lower().str.strip()
df_excel["status"] = df_excel["status"].str.lower().str.strip()
```

#### Анатомический разбор роли каждого символа:
*   **`df_excel["cargo"] =`** — Жесткое «Правило Ювелира». Принудительное переприсваивание. Без него зачистка произойдет «в воздух», а колонка в RAM останется грязной.
*   **`.str`** — Специализированный векторный шлюз Pandas. Он дает команду: «Применяй следующие текстовые инструменты не к одной строке, а ко всему столбцу одновременно в обход циклам `for`».
*   **`.lower()`** — Метод перевода всех букв в нижний (маленький) регистр. Уничтожает разницу между «З» и «з».
*   **`.strip()`** — Метод-триммер. Физически срезает с левого и правого краев текстовых строк все паразитные невидимые пробелы.

### Шаг 5. Точечная заварка сквозных дыр `NaN`

```python
df_excel["status"] = df_excel["status"].fillna("не указан")
df_excel["number"] = df_excel["number"].fillna(0)
```

#### Анатомический разбор роли каждого символа:
*   **`fillna(...)`** — Промышленный инструмент заполнения системных пустот (близнец `None` в Python). Он находит маркеры `NaN` и точечно заменяет их на то, что передано в скобках.
*   **`("не указан")`** — Строковый аргумент. Безопасный текстовый заглушка для строковой колонки `status`.
*   **`(0)`** — Числовой аргумент. Безопасный цифровой ноль, которым мы временно ликвидируем дыру в номерах вагонов, чтобы не сломать математический класс колонки.

### Шаг 6. Межклассовая переплавка и уничтожение паразитной точки

```python
df_excel["number"] = df_excel["number"].astype(int)
df_excel.head()
```

#### Анатомический разбор роли каждого символа:
*   **`.astype(...)`** — Жесткий синтаксический пресс переплавки типов.
*   **`int`** — Целевой класс данных (Integer). Перевод колонки из дробного типа `float64` (появившегося из-за дыр `NaN`) обратно в чистые целые числа `int64`. 
*   **Результат в RAM:** Паразитные точки с нулями (`5501.0` \(\rightarrow\) `5501`) полностью уничтожены. Уникальные бортовые номера вагонов стали эталонными и готовы к сквозным сшивкам.
